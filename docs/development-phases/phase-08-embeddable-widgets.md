# Phase 08 — Embeddable Widgets

## Purpose

Build or prototype the first useful embeddable widget capability if there is a strong use case.

Embeddable widgets are a medium-priority platform capability because they can spread brand recognition.

## Possible widget use cases

- AI Facts / transparency nutrition label.
- Public mini report card.
- Sounds Like Us guidance badge.
- Generated-with-guidance badge.
- Benchmark/report snippet.

## Out of scope

- Making web components the default UI model.
- Widget work without a clear public use case.
- Overbuilding a widget platform before there is a first widget.

## Acceptance criteria

- A first widget use case is selected or the phase is deferred with rationale.
- Widget architecture does not depend on private app-only UI assumptions.
- Public rendering/security implications are documented.
