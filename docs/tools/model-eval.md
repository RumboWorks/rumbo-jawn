# Model Eval

## Status

Planned sibling tool.

Out of scope for the initial Sounds Like Us MVP implementation unless a phase file explicitly says otherwise.

## Why it appears in planning

Model Eval is included in planning so the Rumbo platform does not become a one-off Sounds Like Us app.

Future Model Eval needs influence shared architecture decisions such as:

- centralized auth,
- shared organizations,
- shared billing/usage limits,
- shared AI provider wrapper,
- shared AI call cost logging,
- shared jobs,
- shared storage/artifact manifests,
- centralized admin,
- shared design-system conventions.

## Future product direction

Model Eval may support structured evaluation of AI model outputs against rubrics, criteria, reviewers, scores, artifacts, and reports.

Do not implement Model Eval features during the first MVP unless a phase is revised to include them.
