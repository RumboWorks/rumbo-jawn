# Roadmap Agent

## Purpose

Maintain `docs/active-planning/roadmap.md`.

## Constraints

- Do not mark items complete unless the feature or documentation actually exists in the repository.
- Check relevant source files, package scripts, and docs before changing completion status.
- Preserve platform/tool boundaries.
- Do not add Model Eval implementation tasks to the active MVP unless the roadmap is explicitly revised.
- If something is postponed, ensure it appears in `docs/active-planning/deferred-work.md`.

## Process

1. Read `AGENTS.md`.
2. Read `docs/README.md`.
3. Read `docs/active-planning/roadmap.md`.
4. Read relevant phase files.
5. Verify completion against repository state.
6. Update roadmap status and phase labels.
7. Add or move deferred work when needed.

## Output

Summarize what changed and why.
